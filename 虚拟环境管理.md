# TaskMate 虚拟环境管理指南

## 概述

本文档介绍如何管理 TaskMate 项目的 Python 虚拟环境。

## 虚拟环境位置

```
taskmate/venv/
```

## 常用操作

### 激活虚拟环境

**Windows:**
```cmd
cd taskmate
venv\Scripts\activate
```

**PowerShell:**
```powershell
cd taskmate
.\venv\Scripts\Activate
```

**Linux/Mac:**
```bash
cd taskmate
source venv/bin/activate
```

### 安装依赖

激活虚拟环境后，运行：

```bash
pip install -r requirements.txt
```

### 升级依赖

```bash
pip install --upgrade -r requirements.txt
```

### 导出当前环境的依赖列表

```bash
pip freeze > requirements.txt
```

### 停用虚拟环境

```bash
deactivate
```

## 重新创建虚拟环境

如果需要重新创建虚拟环境：

1. 删除现有虚拟环境目录：
```cmd
rmdir /s /q venv
```

2. 创建新的虚拟环境：
```cmd
python -m venv venv
```

3. 激活并安装依赖：
```cmd
venv\Scripts\activate
pip install -r requirements.txt
```

## 虚拟环境结构

```
venv/
├── Include/          # 头文件
├── Lib/              # 库文件
│   └── site-packages/  # 安装的包
├── Scripts/          # 可执行脚本 (Windows)
│   ├── activate
│   ├── pip.exe
│   └── python.exe
└── pyvenv.cfg        # 虚拟环境配置
```

## 注意事项

- 请勿将 `venv` 目录提交到版本控制系统
- 确保 `.env` 文件中的 API 密钥配置正确
- 首次运行前需要配置 DeepSeek API Key 和 Todoist API Token
